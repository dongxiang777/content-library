# 业务逻辑文档

> 这个项目是干什么的、数据怎么流、做了哪些优化。配合《部署文档》一起看：那篇讲"怎么装起来"，这篇讲"它怎么运转"。

---

## 一、一句话说明

这是一个**短视频爆款文案采集库**：自动从抖音、视频号抓取点赞高的同类爆款视频，转成文字文案，AI 打好分类标签，存进飞书表格，作为我们视频号矩阵账号的"内容弹药库"。

服务于三条业务线，分别对应飞书表格里三个子表：

| 业务线 | 内容方向 | 飞书子表 |
|--------|----------|----------|
| 传承 | 遗嘱 / 意定监护 / 老人赡养 / 继承 | 传承 IP |
| 情感 | 老年情感 / 养老 / 家庭矛盾 | 情感 IP |
| 理赔 | 车险 / 农险 / 保险理赔 | 理赔 |

## 二、两条采集链路

### 链路 A：抖音

- **工具**：MediaCrawler（第三方，`tools/MediaCrawler/`，需单独克隆）。
- **入口**：`scripts/pipeline.py`，支持两种采集方式：
  - 关键词搜索：`crawl --type search --keywords "遗嘱怎么写,自书遗嘱"`
  - 创作者主页：`crawl --type creator --creator-id "主页URL或sec_uid"`
- **特点**：能拿到搜索/主页视频列表及互动数据，按关键词或博主采集。

### 链路 B：视频号

- **工具**：本地 MITM 代理工具（`tools/wx_channels_download/`，源码已并入本仓库）。它拦截微信电脑版视频号的流量，提供本地 API（`127.0.0.1:2022`）。
- **入口**：`scripts/channels/channels_collector.py` + 各账号采集脚本（如 `run_baidajie.py`、`run_homan.py`）。
- **能力**：
  - 按关键词搜账号（`search_accounts`），并按博主昵称二次过滤（如搜"白大姐"只留名字含"白大姐"的号）。
  - 拉取账号主页视频列表（`collect_from_username`），**列表本身就带点赞/收藏/转发/评论数**。
  - 按点赞数前置过滤（`min_likes`，如只采集点赞 ≥ 2000 的）。
  - DRM 解密下载视频 → 抽音频 → 转写。
- **矩阵批量采集**：一个关键词搜出多个账号，逐号采集、自动去重、失败账号跳过留待二轮补跑。

## 三、完整流程（数据怎么流）

不管抖音还是视频号，后半段是同一条流水线：

```
采集视频列表（带互动数据）
   │  按点赞数过滤、去重
   ▼
下载视频（抖音 curl / 视频号 DRM 解密）+ ffmpeg 抽 16kHz 单声道 WAV
   ▼
FunASR 语音转文字（本地推理，支持说话人分离）
   ▼
DeepSeek AI 分类（8 维度：业务方向/细分领域/内容形式/选题/角度/人群/情绪钩子/标签）
   ▼
按"业务方向"路由写入对应飞书子表（传承/情感/理赔）
```

**关键设计：并行流水线**。下载+抽音频（预取）、FunASR 转写（串行，因模型单通道）、AI 分类+写飞书（异步）三段重叠执行，互不等待，整体提速 30–50%。

## 四、做了哪些调整和优化（重点）

按重要性排列：

**1. 视频号列表级互动数据（最关键的突破）**
原来拿不到主页视频的点赞数，只能逐条点进详情页请求，慢且容易触发限制。排查发现：微信原始响应里**本来就带**点赞/收藏/转发/评论数，是 Go 工具的数据结构漏定义了这些字段，被静默丢弃。补全字段后，列表接口一次返回 15 条视频的全部互动数据，**无需逐条请求**，采集效率大幅提升，也让"按点赞过滤"成为可能。

**2. 点赞数前置过滤（`min_likes`）**
在拉列表时就按点赞数筛掉低热度视频（如 < 2000 直接不要），避免下载、转写一堆没价值的视频，省时省盘。

**3. 多子表自动路由 + 强制路由（`force_route`）**
按业务方向把数据写进对应子表（传承/情感/理赔）。新增 `force_route` 参数：明确知道某批数据属于哪条线时（如白大姐→情感），强制按指定子表写，不被 AI 分类结果带偏。

**4. FunASR 长驻进程 + 说话人分离**
模型约 2GB，原来每次转写都重新加载很慢。改成长驻 daemon，模型只加载一次，后续转写走进程通信。同时加了 `cam++` 说话人分离：单人视频返回纯文本，多人连麦视频自动按【说话人1】【说话人2】分段。

**5. 动态列映射**
飞书表格列位置不再写死，运行时读表头自动匹配列号。表格加列、调顺序都不会让脚本崩。

**6. 任务后自动清理视频**
转写完成的视频自动删除（只删有文案的），避免几百个视频堆满硬盘（一次能省 10GB+）。临时音频文件也自动清理。

**7. 跨平台路径修复**
原来配置里写死了我这台 Mac 的绝对路径，换电脑必崩。改成根据项目位置动态计算，并兼容 Windows 的虚拟环境路径格式（`Scripts\python.exe`）。这是"能在别的电脑部署"的基础。

**8. 健壮性加固**
Ctrl+C 安全退出（不丢已采集数据）、原始数据落盘备份、飞书请求重试、失败账号容错跳过、HTML 高亮标签清洗（避免标题残留碎片）等。

## 五、数据字段（飞书表 19 列）

| 列 | 字段 | 说明 |
|----|------|------|
| A | 业务方向 | 传承 / 情感 / 理赔 |

| D | 选题方向 | 自书遗嘱教程 / 独生子女继承保护 等 |

| P | 收藏 | 收藏数 |
| Q | 转发 | 转发数 |
| R | 评论 | 评论数 |
| S | 入库日期 | YYYY-MM-DD |

## 七、常用命令速查

```bash
# 抖音：搜索采集 + 全流程
cd scripts
python3 pipeline.py full --type search --keywords "遗嘱怎么写" --count 20

# 抖音：采集指定创作者
python3 pipeline.py crawl --type creator --creator-id "创作者主页URL"

# 视频号：白大姐矩阵（点赞≥2000 → 情感表）
python3 scripts/channels/run_baidajie.py

# 视频号：Homan校长矩阵
python3 scripts/channels/run_homan.py
```

> 在 QoderWork 里也可以直接用自然语言驱动（`douyin-content-pipeline` 技能），例如："搜'遗嘱怎么写'，点赞 1 万以上，最近一周，来 20 条"。
