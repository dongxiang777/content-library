# 部署文档

> 目标：在一台全新电脑（macOS 或 Windows）上，用最少的步骤把整个内容库跑起来；并且后续在 Mac 上迭代后，其他电脑能一键同步到最新版。

---

## 一、先理解：这个项目分两部分

正如你说的，项目天然分成两块，部署思路也围绕这两块展开：

**第一块：代码和程序（跟着 GitHub 走，所有电脑一样）**

- Python 采集/转写/分类/写入的全部脚本（`scripts/`）。
- 视频号下载工具的**源码**（`tools/wx_channels_download/`，已并入本仓库）。
- 这部分全部存在 GitHub 上，任何电脑 `git clone` / `git pull` 就能拿到，且永远一致。
- 注意：可执行程序（二进制）**不放进仓库**，因为 Mac 和 Windows 的程序互不通用，且每次迭代重新编译会让仓库越来越大。正确做法是每台电脑从同一份源码本地编译（脚本会自动做）。

**第二块：本机依赖和配置（每台电脑各自装一次）**

- Python 虚拟环境 + 依赖库（FunASR 语音识别约 2GB、MediaCrawler 抖音爬虫等）。
- FunASR 模型文件（约 2GB，首次自动下载）。
- 视频号工具的证书安装、代理配置（首次运行自动处理，Windows 需点管理员确认）。
- `.env` 凭证文件（飞书、DeepSeek 的密钥，**不能**放 GitHub）。
- 这部分由一键脚本自动安装/配置，只有密钥需要手动填。

## 二、前置要求

| 项目 | macOS | Windows |
|------|-------|---------|
| Go（编译视频号工具） | `brew install go`（脚本会自动装） | 从 https://go.dev/dl/ 装 .msi，或脚本用 winget 自动装 |
| Python 3.10+ | 一般已有 | 从 https://python.org 安装，勾选"Add to PATH" |
| Git | 一般已有 | 从 https://git-scm.com 安装 |
| 微信电脑版 | 需要（视频号采集用） | 需要（视频号采集用） |
| 网络 | 需能访问 GitHub、ModelScope、pypi 清华镜像 | 同左 |

> 只采集抖音、不采集视频号的话，可以不装 Go、不装微信电脑版。

## 三、一键部署

### macOS

```bash
git clone https://github.com/dongxiang777/content-library.git "content library"
cd "content library"
bash scripts/setup/setup_mac.sh
```

### Windows

```powershell
git clone https://github.com/dongxiang777/content-library.git "content library"
cd "content library"
powershell -ExecutionPolicy Bypass -File scripts\setup\setup_windows.ps1
```

脚本会依次自动完成 7 步：检查/安装 Go → 编译视频号工具 → 克隆 MediaCrawler → 建虚拟环境 → 装依赖（含 FunASR、Playwright）→ 预下载语音模型 → 创建 `.env`。已完成的步骤会自动跳过，**可放心重复运行**。

## 四、部署后手动完成 3 件事

1. **填密钥**：编辑项目根目录的 `.env`（脚本已从 `.env.example` 创建），填入：
   - `FEISHU_APP_ID` / `FEISHU_APP_SECRET`：飞书开放平台（https://open.feishu.cn）→ 你的应用 → 凭证与基础信息。
   - `LLM_API_KEY`：DeepSeek 的 API key。
2. **加协作者**：把飞书应用对应的机器人，加为目标电子表格的协作者（编辑权限），否则写不进去。
3. **采集视频号时**：启动下载工具，然后在微信电脑版打开视频号页面（建立 socket 连接）：
   - macOS：`cd tools/wx_channels_download && ./wx_video_download`
   - Windows：`cd tools\wx_channels_download ; .\wx_video_download.exe`（以管理员身份，首次会弹 UAC 装证书，点"是"）

## 五、日常迭代与多机同步（重点）

这是你最关心的"我在 Mac 上改，别的电脑怎么快速跟上"。

**你在 Mac 上迭代（开发机）：**

```bash
# 改完代码后
git add <改动的文件>
git commit -m "说明这次改了什么"
git push origin main
```

**其他电脑同步（使用机）：**

```bash
cd "content library"
git pull origin main
```

然后判断要不要重跑安装脚本：

| 这次改动涉及 | 同步后需要做什么 |
|--------------|------------------|
| 只改了 Python 脚本 / 文档 | `git pull` 即可，直接能用 |
| 改了视频号工具源码（`tools/wx_channels_download/`） | `git pull` 后重跑一次编译：`bash scripts/setup/build_tool.sh`（Windows 跑 `build_tool.bat`） |
| 加了新的 Python 依赖 | `git pull` 后重跑一次 `setup_mac.sh` / `setup_windows.ps1`（会自动只装缺的） |

> 一句话：**代码改动 `git pull` 就够；动了工具源码就多跑一次编译；动了依赖就多跑一次安装脚本。** 安装脚本是可重复运行的，多跑不会出错。

## 六、视频号工具的证书与代理（本机配置说明）

视频号采集靠一个本地 MITM 代理工具（已并入 `tools/wx_channels_download/`）：

- 它在本地开两个端口：API 服务 `127.0.0.1:2022`、代理 `127.0.0.1:2023`。
- 首次运行自动安装根证书（用于解密视频流量）。Windows 需管理员权限并手动信任证书。
- 代理用选择性 PAC：只有微信/腾讯域名走 MITM，其他流量正常上网（配置在工具的 `config.yaml`，默认上游代理 `http://127.0.0.1:7897`，如你本机科学上网端口不同，改这里）。
- **每次重启工具后，要在微信电脑版重新打开视频号页面**，否则采集会报"请先初始化客户端 socket 连接"。采集前可先 `curl http://127.0.0.1:2022/api/status` 确认 `channels.available=true`。

## 七、常见问题

**Q：编译视频号工具报"请使用 go build -tags embed_inject"？**
A：必须带 `embed_inject` 标签编译。直接用 `scripts/setup/build_tool.sh`（已带），别手动 `go build`。

**Q：转写时报"未安装 funasr"或找不到模型？**
A：确认用的是 MediaCrawler 的虚拟环境（`tools/MediaCrawler/.venv`），并重跑安装脚本预下载模型。模型下载需直连 ModelScope（脚本会自动关代理）。

**Q：视频号采集报"请先初始化客户端 socket 连接"？**
A：在微信电脑版重新打开视频号页面即可（见第六节）。

**Q：写入飞书失败 / 403？**
A：检查 `.env` 凭证是否正确，以及飞书机器人是否已被加为表格协作者（编辑权限）。

**Q：Windows 上证书装不上？**
A：以管理员身份运行工具；如仍失败，证书文件在工具目录下，可手动导入到"受信任的根证书颁发机构"。
